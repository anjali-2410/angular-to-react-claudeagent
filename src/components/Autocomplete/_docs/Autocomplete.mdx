  ## Autocomplete

  The `<Autocomplete>` component provides an intuitive way for users to search and select from a list of options. It displays filtered suggestions as the user types, improving the user experience by reducing the need to scroll through long lists.

  ## Guidelines

  - Use autocomplete when users need to select from a large list of options (typically more than 10 items).
  - Set a minimum search text length to avoid overwhelming users with too many results at once.
  - Provide clear placeholder text to guide users on what they can search for.
  - Use autocomplete for search functionality where users may not know the exact term they're looking for.
  - Consider showing a loading indicator for backend searches that may take time.
  - Display a "No items found" message when search returns no results to provide feedback.
  - Use custom templates when additional context (like paths or descriptions) helps users identify the correct option.
  - The autocomplete panel automatically positions itself based on available space.
