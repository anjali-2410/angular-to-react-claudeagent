# Label

The Label component is a fundamental UI element designed to provide accessible labeling for form controls across applications. It offers consistent, accessible identification of form fields while supporting required indicators, help text, and various interaction states (disabled, read-only).

<hr />

## Guidelines

- Always associate labels with form controls using `labelFor` prop
- Use clear, descriptive label text that explains what the field is for
- Include required indicators (*) for mandatory fields
- Provide help text for complex or unfamiliar fields
- Ensure labels are concise but descriptive
- Maintain consistent labeling patterns across your application

### State-Based Behavior

- **Disabled state**: When `disabled` is true, both required indicators and help text are automatically hidden
- **ReadOnly state**: When `readonly` is true, required indicators are hidden but help text remains visible
- **State restoration**: When returning from disabled/readonly to normal state, indicators are automatically restored

## Do's and Don'ts

### ✅ Do

- Always associate labels with form controls using `labelFor`
- Use clear, descriptive label text
- Include required indicators for mandatory fields
- Provide help text for complex fields
- Keep labels concise but informative

### ❌ Don't

- Use vague or generic label text
- Forget to connect labels to their form controls
- Overuse help text for obvious fields
- Make labels too long or wordy
- Use placeholder text as a replacement for labels

<hr />

## Use Cases

### Form Fields

Use labels for all form inputs to ensure accessibility and clarity. Always associate the label with its corresponding input using the `labelFor` prop.

### Required Fields

Mark required fields with the required indicator to help users understand which fields are mandatory before submitting forms.

### Complex Fields

Provide help text for fields that might need additional explanation, such as password requirements, formatting guidelines, or field purposes.

### Disabled States

Apply disabled styling to labels when their associated form fields are disabled to maintain visual consistency.
