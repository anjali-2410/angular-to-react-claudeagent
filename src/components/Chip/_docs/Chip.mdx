# Chip

The `Chip` component displays a compact element that represents an attribute, tag, or action. Chips can include an icon, a label, and an optional remove action.

---

## Guidelines

- Use chips to represent short, discrete pieces of information such as tags, selected filters, or compact actions.
- Keep chip labels short; when text is longer than available width the label will be truncated and a tooltip will expose the full text on hover and keyboard focus.
- Use `errorState` to indicate validation or status problems.
- Prefer `small` size for inline lists; `medium` is available when more visual prominence is required.

### Accessibility

- The chip is keyboard-focusable via `chipTabIndex`.
- `ArrowRight` from the chip body moves focus to the close button (when `allowClose` is true).
- `ArrowLeft` from the close button returns focus to the chip body.
- The close button has an accessible `aria-label` via `closeIconAriaLabel` (default: `'Remove'`).
