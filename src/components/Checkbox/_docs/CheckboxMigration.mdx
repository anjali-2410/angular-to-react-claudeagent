# Checkbox — Angular → React Migration Guide

## Import

```ts
// Angular
import { CheckboxModule } from '@niceltd/sol/checkbox';

// React
import { Checkbox, CheckboxGroup } from '@niceltd/sol-react/checkbox';
```

## Controlled vs uncontrolled

Angular two-way binding maps to the React controlled pattern:

```tsx
// Angular
<sol-checkbox [(checked)]="myValue" (checkedChange)="onChecked($event)" />

// React — controlled
<Checkbox checked={myValue} onCheckedChange={e => setMyValue(e.checked)} />

// React — uncontrolled (manages its own state)
<Checkbox onCheckedChange={e => console.log(e.checked)} />
```

## CheckboxEvent vs CheckboxChangeEvent

The Angular `CheckboxEvent` carried a reference to the source component. The React `CheckboxChangeEvent` contains only the data:

```ts
// Angular
export class CheckboxEvent {
  source: CheckboxComponent;
  checked: boolean;
  indeterminate: boolean;
}

// React
interface CheckboxChangeEvent {
  checked: boolean;
  indeterminate: boolean;
}
```

## Breaking changes from Angular SOL 14 → SOL 16

- `readOnly` was renamed to `readonly` in SOL 16 — the React prop is `readonly`.

## Forms integration

Angular `ngModel` / `ReactiveFormsModule` integration does not apply in React. Use the controlled pattern with `useState`:

```tsx
const [accepted, setAccepted] = useState(false);

<Checkbox
  checked={accepted}
  onCheckedChange={e => setAccepted(e.checked)}
  label="I accept the terms"
/>
```

For `CheckboxGroup`:

```tsx
const [selections, setSelections] = useState<string[]>([]);

<CheckboxGroup
  value={selections}
  onClicked={setSelections}
  label="Email Preferences"
>
  <Checkbox value="marketing">Marketing Emails</Checkbox>
  <Checkbox value="updates">Product Updates</Checkbox>
  <Checkbox value="alerts">Security Alerts</Checkbox>
</CheckboxGroup>
```
