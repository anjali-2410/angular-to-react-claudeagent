### Types

```ts
type CalendarType =
  | 'single-selection'
  | 'multi-single-selection'
  | 'week-selection'
  | 'multi-week-selection'
  | 'range-selection';

interface DateRange {
  start: Date | null;
  end: Date | null;
}

type CalendarValue = Date | Date[] | DateRange | DateRange[];

interface SpecialDates {
  date: Date;
  status: 'partial' | 'full';  // 'partial' = dot, 'full' = filled dot
}
```

**Value type by selection mode:**

| `type` | `value` / `onChange` payload |
| --- | --- |
| `single-selection` | `Date` |
| `multi-single-selection` | `Date[]` |
| `week-selection` | `DateRange` |
| `multi-week-selection` | `DateRange[]` |
| `range-selection` | `DateRange` |

## Angular → React Migration

| Angular `@Input` / `@Output` | React prop | Notes |
| --- | --- | --- |
| `[type]` | `type` | Same values |
| `[daysDisabled]` | `daysDisabled` | Same |
| `[displayWeeks]` | `displayWeeks` | Same |
| `[startingDay]` | `startingDay` | Same |
| `[min]` | `min` | Same |
| `[max]` | `max` | Same |
| `[specialDates]` | `specialDates` | Same |
| `[ariaDescribedby]` | `ariaDescribedby` | Same |
| `[(ngModel)]` | `value` + `onChange` | Controlled pattern |
| `(dateSelected)` | `onDateSelected` | Same payload |

**Angular `DateRange` vs React `DateRange`:**

Angular used Angular Material's `DateRange<Date>` with `start` / `end`. React uses a plain object `{ start: Date | null, end: Date | null }` — same property names, no library dependency.

**`ngModel` → controlled pattern:**

```tsx
// Angular
<sol-calendar [(ngModel)]="selectedDate" type="single-selection" />

// React
const [date, setDate] = useState<Date | undefined>();
<Calendar type="single-selection" value={date} onChange={v => setDate(v as Date)} />
```

## Migration to v16

**SpecialDates interface:** Property `color` was renamed to `status` in v16. Use `'partial'` or `'full'`.
