# Dropdown: Migrating from Angular to React

## Import

```tsx
// Angular
import { DropdownModule } from '@niceltd/sol/dropdown';

// React
import { Dropdown } from './components/Dropdown';
```

## Selector

```html
<!-- Angular -->
<sol-dropdown ...></sol-dropdown>

<!-- React / Next.js -->
<Dropdown ... />
```

## Controlled Component Pattern

Angular used `ControlValueAccessor` for form integration. In React, use the controlled pattern:

```tsx
// Angular (template-driven)
<sol-dropdown [(ngModel)]="selection" [options]="options"></sol-dropdown>

// React controlled
const [value, setValue] = useState(null);
<Dropdown value={value} options={options} onSelectionChange={setValue} />
```

## React Hook Form Integration

```tsx
import { useForm, Controller } from 'react-hook-form';

const { control } = useForm();

<Controller
  name="color"
  control={control}
  rules={{ required: true }}
  render={({ field }) => (
    <Dropdown
      options={options}
      value={field.value}
      onSelectionChange={field.onChange}
      onBlur={field.onBlur}
      required
    />
  )}
/>
```

## CDK Overlay → @floating-ui/react

The Angular version used `@angular/cdk/overlay` for positioning. The React version uses `@floating-ui/react` (`useFloating`) which provides the same positioned-overlay behavior.

## Template Refs → Render Functions

```tsx
// Angular
<ng-template #myTemplate let-opt>
  <span>{{ opt.name }}</span>
</ng-template>
<sol-dropdown [optionTemplate]="myTemplate"></sol-dropdown>

// React
<Dropdown
  optionTemplate={(opt) => <span>{opt.name}</span>}
/>
```

## Next.js — `'use client'`

Both `Dropdown` and `DropdownTrigger` use React hooks and event handlers. They require the `'use client'` directive (already included). If you use them in a Next.js Server Component file, wrap in a Client Component boundary.

## Migration from Breeze

For changes from the Breeze (CXone) dropdown components, see the [original Angular migration guide](https://github.com/nice-cxone/sol-components) for input/output rename details. The React prop names follow the SOL Angular naming (not Breeze).
